Element-UI 组件使用总结

刚入职接触的都是公司项目的后台，两个项目的后台的前端都是使用 Element-UI 组件库。大部分时候后台的功能就那么几个，把常用的组件在实际项目场景下如何使用给总结好，可以提高工作的效率

表单校验

在操作后台的工作人员填写表单时进行检验，这个涉及到规则的定义。

表单的数据

我们会把表单的数据对象通过一个叫 model 的 props 传入到 <el-form> 这个组件内。这个表单数据对象一般我们会和<el-input> <el-select>等表单控件使用 v-model 来做双向绑定，在提交表单时就将这个数据对象发送给后端

校验的规则

另外，我们还需要传入一个 rules ，它是一个对象。这个对象的的键名是上面model 数据对象中的字段，来规定数据对象中的哪些值需要被校验。

这个rules 对象的值是一个包含一个或多个对象的数组。一个对象就对应着一条校验规则。在这个对象中有个validator的字段，该字段的值为函数名，我们通过这个函数来定义更复杂的验证规则。

该函数的三个参数，rule为规则，value是输入的值，callback是规则不通过的回调。

需要校验的字段是对应哪个表单项

通过前面给 <el-form> 传入model和rules，组件知道表单中的数据是什么、每条数据的校验规则是什么？我们还需要告诉组件，一个数据的校验规则对应哪个表单项。我们把 <el-form-item> 的 prop 属性设置为需校验的字段名即可

监听 Table 组件的 sort-change @sort-change="handleSort"

再给 TableColumn 组件添加 sortable 的属性值为 "custom"。

最后，开始写 handleSort 这个函数。这个示例代码中orderBy 是给后端的字段，这个和后端约定的是在参数为 1 时是升序，为 2 时是降序。在这个函数的参数中可以拿到一个对象，对象中的有一个 order 是表格中此列当前排序的方法（升序或降序）。我们就可以根据此时表格的升序降序来给后端请求不同的参数。

开关

在页面中添加一个开关，用 v-model 来双向绑定一个值。在之前做的时候值用的是 Boolean 类型，也就是打开时是 true，关闭时为 false。但是，如果这个值传个后端的参数。比如需要开关打开时 0，关闭时是 1。Element-UI 提供了一对属性 active-value和inactive-value，用来指定打开和关闭时的值。

如果在关闭和打开时需要做一些事情可以监听 change 事件
