Git 常用命令小本本

目的是记录下，日常开会经常要用到的 git 命令，忘记了就来这里翻小本本。

git init 初始化仓库

  • 正常情况将返回 Initialized empty Git repository 告诉我们，一个新的空仓库为您准备好啦 🎉
  • 如果将一个已经存在的仓库再执行这条命令，将提示 Reinitialized existing Git repository ... 重新初始化已存在的仓库。
  • 将在添加一个 .git 的目录，这个目录是Git来跟踪管理版本库的。

git clone 克隆远程仓库

☁️ — > 💻

克隆，复制一份一模一样的仓库

git add 把文件添加到仓库

可反复多次使用，添加多个文件。

git status 查看仓库的状态

输入这个命令后，会有三种不同状态提示

  1. 所有文件状态和远程仓库相同
  2. 有修改的文件，已经提交到了本地仓库的暂缓区
  3. 有新添加的文件或有被修改的文件，还未被做过任何处理。会提示

git diff 比对已被修改的文件，前后版本的差异

git commit 把文件提交到仓库

  • -m 后面输入一段字符串，是本次提交的说明

git reset

  • 已暂存的文件取回工作区
  • 在分支中，设置指针指向那个版本。

git log & git reflog 查看提交历史

这个可以用与查看各个版本的 <commit_id> 来进行退回操作

git checkout 用版本库里的版本替换工作区的版本

  • 已修改的文件，退回修改之前版本
  • 切换当前分支

git pull

git remote 标签管理

add 命令：添加一个远程库的标签。可以方便的向不同的远程仓库来push
set-url 命令：修改某个远程仓库标签对于的地址

git push 推送到远程仓库

-f 参数：强制推送，会覆盖远程仓库中的代码

git branch 新增 & 查看分支操作

git merge 合并某分支到当前分支

如何修复冲突

假设的情况是这样子：大家共同开发一个项目，同事那边先修改了文件，并且push到了远程仓库。我在他之后，把我本地仓库一相同文件也修改了，当我再进行push时到远程仓库，Git 就会报错。因为有两个修改，它不知道要如何来决定。

需要先 pull，然后git会把需要解决冲突的地方标记。让自己手动进行一个合并

分支的意义


Git分支十分强大，在团队开发中应该充分应用。

git 目录下的文件的三种状态

  • 已提交（mommitted）
  • 已修改（mofified）
  • 已暂存（staged）

git 暂存区概念的理解

git add命令实际上就是把要提交的所有修改放到暂存区（Stage），然后，执行git commit就可以一次性把暂存区的所有修改提交到分支。
