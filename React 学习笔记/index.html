React 出现的历史背景

Facebook 的前端工程师傅们在写功能时遇到 BUG，他们分析传统的 Web UI 开发存在的问题：

  1. 传统 UI 操作关注太多细节
  2. 应用程序状态分散在各处，难以追踪和维护

那么 React 是如何来解决这两个问题呢？

  1. 在状态改变时 React 帮我们做整体的刷新，来解决 UI 操作关注太多细节。
  2. 传统 MVC 结构有非常多的 Model 和 View，这些 Model 和 View 的使用关系错综复杂、而且是双向绑定的。React 提出了 Flux 架构（单向数据流的设计模式）
